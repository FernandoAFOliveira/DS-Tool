PriorityQueue

PriorityQueue stores elements so that the highest-priority or lowest-priority item can be removed first. PriorityQueue does not keep all elements fully sorted. It only guarantees that peek() and poll() access the next priority item. Use PriorityQueue for scheduling tasks, processing jobs by priority, or algorithms that repeatedly need the next smallest item.

API Overview

add(E e), offer(E e), poll(), peek(), remove(Object o), size()

Key idea: Iterating over a PriorityQueue does NOT give sorted order. To process elements in priority order, you must repeatedly call poll(). 
PriorityQueue<Integer> pq = new PriorityQueue<>();

pq.add(5);
pq.add(2);
pq.add(8);

// Access highest priority (smallest)
System.out.println(pq.peek()); // 2

// Remove in priority order
while (!pq.isEmpty()) {
    System.out.println(pq.poll()); // 2, 5, 8
}

// ⚠️ Common mistake: iteration is NOT sorted
pq.add(5);
pq.add(2);
pq.add(8);

for (Integer x : pq) {
