Queue

Queue represents first-in, first-out behavior, where the first item added is the first item removed.

Use Queue for waiting lines, task processing, breadth-first search, or anything processed in arrival order.

API Overview

add(E e), offer(E e), poll(), peek(), remove(), size()

Key idea: Queue = First-In, First-Out (FIFO). Always remove from the front and add to the back. Queue is an interface — use LinkedList or ArrayDeque as implementations.
Front → [Alice, Bob, Charlie] → Back 
Queue<String> queue = new LinkedList<>();

// Add to the back
queue.offer("Alice");
queue.offer("Bob");

// Peek (look at front without removing)
System.out.println(queue.peek()); // Alice

// Remove from the front (FIFO)
System.out.println(queue.poll()); // Alice
System.out.println(queue.poll()); // Bob

// ⚠️ Difference between methods
// offer() / poll() → safe (return null if empty)
// add() / remove() → throw exception if empty
